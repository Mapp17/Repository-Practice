# Good User Story
As a registered user, I want login in to the system, so that can access my dashboard.

# Acceptance Criteria
- User can enter email and password.
- if creddentials are invalid, an error is shown.
- Successfuul logon redirect to dashboard.

# Comparing vague vs testable friendly.

## Vague
The system must be user friendly.
### Subjective
### Not testable
### High pontial for conflict

## Testable
